虚拟机的搭建过程

第一次接触虚拟机这个东西，跟着网上的教程，少走了很多弯路，由于是第一次搭建不太熟悉，但还是完成了任务。

一、下载虚拟机运行软件以及Ubuntu桌面系统
我这边准备下载的是VMware Workstation Pro 17，也是目前 Windows 最主流的虚拟机软件。
分别在其官网上下载好之后，准备开始安装。

二、安装虚拟机
根据提示慢慢走就好了，许可证之类的东西可以在网上找，很多都能用。

三、创建Ubuntu 系统虚拟机
1.打开刚刚安装好的 VMware Workstation Pro 17，进入软件后，点击“创建新的虚拟机”。
2.选择“自定义（高级）”
3.选择“安装程序光盘映像文件”，点击输入框旁边的“浏览”按钮，根据下载路径，找到下载好的 Ubuntu 镜像安装包并选择。
4.如果能看到标注的小字（已检测到Ubuntu 64位 22.04.4），说明安装包是对的，并且可以正常安装，这时点击“下一步”。
5.根据网上的教程，他推荐我每个处理器的内核数量修改成2，内存选择4GB，以及网络类型选择桥接，还有硬盘容量我选择了60GB因为的我的盘符还有180G，我就弄多点了，自此我们把 Ubuntu 的虚拟机创建好了，但还没安装此系统，正式的安装需要启动虚拟机。

四、Ubuntu 系统安装过程的配置
1.虚拟机创建好后，进入虚拟机。
2.本着学习的态度，我选择了全英文的开发环境。
3.之后就是next + install now + continue一路过，时区选择shanghai
4.安装完成后 Restart Now 重启
5.初次进入 Ubuntu 系统会有一个欢迎界面，内容忽略，直接点击右上角的“Skip”跳过。
这里是提示是否要升级成 Ubuntu Pro，根据网上的教程建议默认选择“Skip for now”，然后点击“Next”。然后是 Ubuntu 系统的改善建议，改成“No, don’t send sysytem info”，然后点击“Next”。最后一步直接“Done”跳过，自此Ubuntu22.04虚拟机搭建完毕。

搭建博客的心路历程

第一次使用Github+hexo搭建博客，在CSDN和b站上看了一些的教程，其中也走了一些弯路，第一次搭建可能不太熟悉之前这些工具也没使用过但是搭建完成还是有那么一点成就感的。
一、准备工作：安装了本次搭建博客需要用到的软件
1.根据前人指路，我需要安装node.js和Git
两者安装完成之后，在系统变量下的Path可以看到两者已自动修改了环境变量，这时我们可以通过cmd命令查看两者版本。(确认是否安装成功及可以使用)
查看Node.js版本命令：node -v
查看Git版本命令：git –version

二、注册Github账号以及建立仓库
这里就是跟着教程走，由于不是国内的网站，有时候会进不去，还是比较烦的。
这里需要注意的点是仓库的取名格式是：用户名.github.io（这是我们以后的访问域名）
例如我的访问域名就是 moonesy.github.io

三、接下来我进入本地配置安装Hexo
1.在磁盘中创建一个用来存放Github本地仓库文件的目录(可自定义目录存放，顾名思义blog==博客)
选中创建的目录 右键选择使用Git Bash Here打开Git命令窗口
2.正式开始安装hexo
输入命令：cnpm install -g hexo-cli
3.之后就是初始化Hexo
输入命令：hexo init
（在这时我遇到了第一个问题，文件无法写入，通过查询网上的方法，发现需要打开文件中的写入权限，打开后问题解决。）
4.这时我们创建的目录下已经多出许多文件
5.此时我们在git bash here中输入命令： hexo s就可以看到我们的hexo页面，接下来根据提示关闭hexo按 Ctrl+C

四、设置ssh
1.输入命令：cd ~/.ssh进入.ssh文件
在这里，我遇到了第二个问题，就是无法找到我的ssh文件，通过查询，我没有这个文件，试了一些方法无果后，我决定按照别人的路径，自己创造一个一样的路径。于是问题解决。
2.此时我按照教程
输入命令：cd ~/.ssh进入.ssh文件
输入命令：ssh-keygen -t rsa -C ‘注册时的邮箱地址’
3.此时我们需要去打开目录中的id_rsa.put文件并复制生成的key，在已登录的Github主页点击Settings设置将我们的密钥key输入
4.这里需要在本地进行验证一次
输入命令：ssh -T git@github.com 并且输入yes之后，行末尾会显示你的用户名。这时我们绑定成功会在邮箱收到文件。
5.接着在本地绑定与Github的用户名和邮箱
输入命令：git config –global user.name “注册时用户名”
输入命令：git config –global user.email “注册时邮箱”

五、上传测试博客
1.用记事本打开并修改本地仓库目录下_config.yml文件
在文件的末尾修改(repository属性的地址就是之前创建仓库后所保存的http地址)
deploy:
type: git
repository: https://github.com/用户名/用户名.github.io.git
branch: master
2.此时需要安装一个上传工具
输入命令：cnpm install hexo-deployer-git
（此时发现命令无效，根据报错原因说明其没有权限，所以我以管理员身份运行，于是问题解决）
3.新建文章
输入命令：hexo new “文章名称”
新建文章之后需要生成一遍文件
输入命令：hexo g
新建文章后可在本地先预览一遍
输入命令：hexo s浏览器输入：localhost:4000(ctrl + c 关闭)
最后直接部署到Github就可以了
输入命令：hexo d
（第一次上传文件成功后到Github仓库邮箱还会收到一封邮件）
然后在Github仓库查看就发现多了很多文件

六、博客搭建成功
1.之后在浏览器中输入：用户名.github.io(Github仓库名)就可以访问博客了。
在这时我遇到了搭建博客时的最后一个问题，就是hexo部署后404以及成功进入我的网页中只有文字，没有主题。于是通过查找网上的各种方法，发现是本地仓库目录下的_config.yml文件出现问题，填写的不是根目录，资源加载不了。
本来是
# URL
## Set your site url here. For example, if you use GitHub Page, set url as ‘https://username.github.io/project‘
url: http://Moonesy.github.io/project
修改为
# URL
## Set your site url here. For example, if you use GitHub Page, set url as ‘https://username.github.io/project‘
url: http://Moonesy.github.io
自此，我的博客搭建完毕。
